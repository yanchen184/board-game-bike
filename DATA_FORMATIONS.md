# 🚴‍♂️ 隊形系統詳細數據

## 隊形系統概述

### 核心機制
- **風阻減免**：隊形的主要優勢，降低體力消耗
- **位置分工**：不同位置承擔不同職責
- **輪替機制**：定期更換領騎位置
- **協同效果**：隊員之間的配合加成

### 隊形切換規則
- 切換時間：需要30秒完成隊形變換
- 切換消耗：消耗5%當前體力
- 冷卻時間：5分鐘內不能再次切換
- 特殊情況：某些事件強制改變隊形

## 二、進階隊形機制

### 動態隊形調整

**自動調整觸發條件**：
1. **爬坡自適應**：坡度>5%時自動散開
2. **逆風應對**：風速>20km/h自動收緊
3. **體力保護**：有人體力<30%自動調整位置
4. **衝刺準備**：最後5km自動準備衝刺隊形

## 六、隊形數值計算公式

### 風阻計算
```javascript
// 基礎風阻
baseWindResistance = speed² * dragCoefficient

// 隊形風阻減免
effectiveWindResistance = baseWindResistance * (1 - formationBonus)

// 位置修正
positionModifier = {
  lead: 1.0,    // 領騎完全承受
  second: 0.8,  // 第二位
  third: 0.75,  // 第三位
  fourth: 0.7   // 第四位
}

// 最終風阻
finalResistance = effectiveWindResistance * positionModifier * weatherFactor
```

### 體力消耗計算
```javascript
// 基礎消耗
baseStaminaUse = distance * effort * terrainFactor

// 隊形修正
formationEfficiency = 1 - (windReduction * 0.8) // 風阻減免的80%轉化為體力節省

// 位置消耗倍率
positionMultiplier = {
  領騎: 1.5,
  側翼: 1.2,
  跟隨: 0.8,
  保護位: 0.6
}

// 最終消耗
finalStaminaUse = baseStaminaUse * formationEfficiency * positionMultiplier
```

### 速度加成計算
```javascript
// 隊形基礎加成
formationSpeedBonus = {
  單人: 1.05,
  雙人: 1.00,
  單線: 1.03,
  火車: 1.05,
  鑽石: 1.04,
  梯隊: 1.10-1.30 // 遞增
}

// 默契度加成
chemistryBonus = 1 + (teamChemistry / 100) * 0.15

// 最終速度
finalSpeed = baseSpeed * formationSpeedBonus * chemistryBonus
```

## 七、隊形戰術應用

### 經典戰術

1. **消耗戰術**
   - 使用火車陣型高速拉扯
   - 目的：消耗對手體力
   - 執行：前60%路程保持高速

2. **保護戰術**
   - 使用鑽石或火車保護主將
   - 目的：確保主將體力充足
   - 執行：主將始終處於保護位

3. **突圍戰術**
   - 單人或雙人快速脫離
   - 目的：打亂對手節奏
   - 執行：關鍵點發動突襲

4. **終點衝刺**
   - 梯隊衝刺隊形
   - 目的：確保勝利
   - 執行：最後10km執行

### 應變策略

| 突發情況 | 當前隊形 | 建議調整 | 原因 |
|----------|----------|----------|------|
| 隊友掉隊 | 火車陣型 | 改為單線 | 人數不足 |
| 強側風 | 單線隊形 | 改為鑽石 | 提高穩定性 |
| 機械故障 | 任意 | 雙人並行 | 方便支援 |
| 體力危機 | 單人 | 火車陣型 | 最大化保護 |

## 總結

隊形系統是遊戲的核心戰術元素，正確的隊形選擇和執行可以節省30%以上的體力消耗，錯誤的隊形可能導致體力過早耗盡。玩家需要根據路況、天氣、隊員狀態動態調整隊形，這種即時戰術決策構成了遊戲的主要策略深度。

最重要的是：**沒有萬能隊形，只有最適合當前情況的隊形**。
